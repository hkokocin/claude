# Commit Message Format

1. Determine the **commit type**: `Feature` (new behaviour), `Fix` (bug fix), or `Chore` (refactor, config, docs, tests).
2. **Title**: `<Type>: <concise imperative description>` (e.g. `Feature: Add SSO login via SAML`).
3. **Body**: 0–5 bullet points covering the most significant changes.
   - Only include bullet points that are genuinely significant. Fewer is better. An empty body is fine for small changes.
   - Write from a product/user perspective, not technical details.
   - Exception: document architectural decisions that impact other parts of the project.
4. Do NOT include `Co-Authored-By` lines.
